# Universal Clipboard — MVP 使用说明

## 概述

一款跨设备剪切板同步工具。复制文本后自动同步到其他设备的剪切板，支持端到端加密。

**当前支持平台：** macOS、Android、Windows
**同步方式：** 腾讯云开发 (CloudBase) 云函数中转
**加密方式：** AES-256-GCM 端到端加密

---

## 前置条件

- 所有设备需联网
- 所有设备输入相同的主密码

---

## macOS 使用

### 首次启动

1. 运行 App
2. 进入解锁页面，输入主密码（如：`my-clipboard-2024`）
3. 点击「创建并开始」
4. 进入主页，显示「暂无剪切板历史」

### 日常使用

- 在任何应用中复制文本 → 自动同步到其他设备
- 主页显示历史记录列表
- 点击条目上的「复制」按钮可重新复制
- 点击「置顶」按钮可置顶常用内容
- 点击「刷新」按钮手动拉取最新内容

### 多选拼接

1. 点击右上角「多选拼接」图标
2. 选择要拼接的条目（按选择顺序编号）
3. 选择分隔符（换行、逗号、分号、空格）
4. 底部实时预览拼接结果
5. 点击「复制拼接内容」

### 系统托盘

- 菜单栏显示 📋 图标
- 点击图标可显示/隐藏窗口
- 右键菜单：显示窗口、退出

## Android 使用

### 安装

```bash
cd ~/universal_clipboard
flutter build apk --release
```

产物路径：`build/app/outputs/flutter-apk/app-release.apk`

传到手机安装（需开启「允许安装未知来源应用」）。

### 首次使用

1. 打开 App
2. 输入与 macOS 相同的主密码
3. 点击「创建并开始」

### 日常使用

- 复制文本 → 自动上传到云端
- 其他设备的复制内容会自动同步到手机剪切板
- 直接在任何应用中粘贴使用

---

## Windows 使用

```bash
cd ~/universal_clipboard
flutter build windows --release
```

产物路径：`build/windows/x64/runner/Release/`

将整个 Release 文件夹复制到 Windows 电脑，运行 `universal_clipboard.exe`。

使用方式与 macOS 相同。

---

## 多设备同步流程

```
1. 设备 A 复制文本
2. App 自动检测变化 → 加密 → 上传到腾讯云数据库
3. 设备 B 轮询到新内容 → 解密 → 写入系统剪切板
4. 设备 B 直接粘贴即可使用
```

**同步延迟：** 约 0.5-1 秒（500ms 轮询间隔 + 网络延迟）

---

## 主密码说明

- 首次使用时设置，之后所有设备必须输入相同密码
- 密码不存储在任何服务器，无法找回
- 忘记密码 = 所有加密数据无法解密
- 建议使用易记但他人难以猜到的密码

---

## 历史记录

- 默认保留最近 100 条
- 可在「设置」中调整（10-500 条）
- 置顶的条目不会被自动清理
- 支持手动删除

## 常见问题

**Q: 同步失败怎么办？**
A: 点击状态栏的「重试」按钮。如果持续失败，检查网络连接和云函数配置。

**Q: 新设备如何加入？**
A: 安装 App → 输入相同主密码 → 自动加入同步。

**Q: 如何退出登录？**
A: 当前版本不支持退出。如需重新登录，删除 App 重新安装。

**Q: 数据存在哪里？**
A: 加密数据存储在腾讯云数据库。本地通过 SharedPreferences 存储历史记录缓存和设备信息。

**Q: 安全性如何？**
A: 所有剪切板内容在设备端加密后才上传。腾讯云只存储密文。即使数据泄露，没有主密码无法解密。
