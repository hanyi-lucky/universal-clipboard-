## v1.2 — 迁移国内服务器 ⭐ 高优先级

**目标：** 替换 Firebase，解决国内需要梯子才能使用的问题

**背景：** Firebase 服务在国内被墙，用户必须挂梯子才能正常使用 App，严重影响可用性。

| 功能 | 说明 |
|-----|------|
| 后端迁移 | 替换 Firebase 为国内可访问的服务 |
| 数据库迁移 | Firestore → 国内云数据库 |
| 认证迁移 | Firebase Auth → 自建认证服务 |
| 存储迁移 | Firebase Storage → 国内对象存储 |
| 数据迁移工具 | 帮助现有用户从 Firebase 迁移数据 |
| 国内网络优化 | 确保同步延迟低于 500ms |

**候选方案：**

| 方案 | 优点 | 缺点 |
|-----|------|------|
| **Supabase（自托管）** | 开源、PostgreSQL、可部署在国内服务器 | 需要自己维护服务器 |
| **LeanCloud** | 国内服务、免梯子、有免费额度 | 第三方依赖 |
| **自建轻量服务** | 完全掌控、无第三方依赖 | 需要服务器和域名 |
| **腾讯云开发** | 国内服务、免梯子、有免费额度 | 腾讯生态绑定 |
| **阿里云函数计算** | 国内服务、按量计费 | 配置复杂 |

**推荐方案：自建轻量 WebSocket 服务**
- 部署在便宜的国内 VPS（如腾讯云轻量、阿里云轻量）
- 使用 WebSocket 实现实时同步（替代 Firestore 轮询）
- 自建认证（JWT token）
- 使用国内对象存储（腾讯 COS / 阿里 OSS）存储图片和文件
- 月成本约 10-30 元

## v1.3 — 自定义账号系统

**目标：** 替换匿名登录，支持多用户

| 功能 | 说明 |
|-----|------|
| 邮箱/密码注册登录 | 替换匿名登录，用户可注册账号 |
| 账号绑定设备 | 每个账号管理多台设备 |
| 设备管理页面 | 查看已绑定设备，支持移除设备 |
| 退出登录 | 支持切换账号 |
| 多用户数据隔离 | 不同账号的剪切板数据完全隔离 |
| 密码找回 | 通过邮箱重置密码 |

**技术方案：**
- 使用自建认证服务（JWT token），基于 v1.2 的国内服务器
- 数据路径改为 `users/{user_id}/...`
- 本地存储账号信息，支持自动登录
